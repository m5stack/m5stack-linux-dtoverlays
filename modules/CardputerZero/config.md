
## Linux Keymap 格式解析

Linux keymap 格式：`0xRRCCKKKK`
- `RR` = Row（行）
- `CC` = Column（列）  
- `KKKK` = Keycode（键码）

---

## `0x04000001` 解析示例

| 字段 | 十六进制 | 十进制 | 含义 |
|------|---------|--------|------|
| RR | `0x04` | 4 | 第4行 |
| CC | `0x00` | 0 | 第0列 |
| KKKK | `0x0001` | 1 | KEY_ESC |

## 总结

- **keymap0**：字母键盘布局（QWERTY），包含方向键
- **keymap1**：数字+功能键布局（F1-F12），包含Home/End/PgUp/PgDn
- **switch-button-code = <73>**：切换按键为 `KEY_F11`(73) 用于在两个keymap之间切换
- **GPIO 27**：中断引脚（输入，上拉）
- **GPIO 17**：表格选择引脚（输出，控制keymap切换）

## 你提到的符号说明

| 符号 | 说明 |
|------|------|
| `'` | `KEY_APOSTROPHE` (单引号) |
| `{ }` | 需要 Shift + `[ ]` 组合输入 |
| `\` | `KEY_BACKSLASH` |
| `\|` | 需要 Shift + `\` 组合输入 |

**注意**：`{ }` 和 `|` 在标准 Linux 键码中没有独立按键，需要通过 `Shift` + 对应按键组合实现：
- `{` = Shift + `[` (KEY_LEFTBRACE + Shift)
- `}` = Shift + `]` (KEY_RIGHTBRACE + Shift)
- `|` = Shift + `\` (KEY_BACKSLASH + Shift)
